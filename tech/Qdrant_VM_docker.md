# Инструкция: Развёртывание Qdrant в Docker с нуля (без прав sudo)

## Предварительные условия

Данная инструкция предполагает, что:

- На сервере уже установлен Docker администратором
- Ваш пользователь уже добавлен в группу docker администратором
- У вас нет пароля для sudo (или он неизвестен)

> Если Docker не установлен или вы не в группе docker — необходимо обратиться к администратору, без sudo это сделать невозможно.

## Шаг 1. Подключение к серверу по SSH

```
ssh ваш_пользователь@<IP_СЕРВЕРА>
```

Пример:

```
ssh dashrim@93.77.191.237
```

При первом подключении подтвердите fingerprint:

```text
Are you sure you want to continue connecting (yes/no/[fingerprint])? yes
```

## Шаг 2. Проверка доступности Docker

Убедитесь, что Docker установлен и работает:

```bash
docker version
```

Ожидаемый результат: вывод информации о клиенте (Client:) и сервере (Server:).

Если появляется ошибка `permission denied` — переходите к Шагу 3.

## Шаг 3. Активация прав группы docker

❗ **Важно:** Если вас только что добавили в группу docker, текущая SSH-сессия об этом не знает.

Выполните команду для запуска новой оболочки с обновлёнными правами:

```bash
newgrp docker
```

Появится стандартное приветствие системы — это нормально.

Проверьте, что права применились:

```bash
docker ps
```

Ожидаемый результат: пустая таблица (или список контейнеров) без ошибки `permission denied`.

## Шаг 4. Проверка свободности порта 6333

Qdrant по умолчанию использует порты 6333 (HTTP API) и 6334 (gRPC).

Проверьте, не заняты ли они:

```bash
ss -tulpn | grep 6333
```

или

```bash
netstat -tulpn | grep 6333
```

Если порт занят:

- Остановите процесс: `pkill -f qdrant`
- Или используйте другие порты (см. раздел «Альтернативные порты» в конце)

## Шаг 5. Создание Docker-тома для хранения данных

Том нужен, чтобы данные Qdrant сохранялись при удалении/обновлении контейнера:

```bash
docker volume create qdrant_storage
```

Проверить, что том создан:

```bash
docker volume ls | grep qdrant
```

## Шаг 6. Запуск контейнера Qdrant

```bash
docker run -d \
  --name qdrant \
  --restart unless-stopped \
  -p 6333:6333 \
  -p 6334:6334 \
  -v qdrant_storage:/qdrant/storage \
  qdrant/qdrant:latest
```

**Пояснение параметров:**

| Параметр | Значение |
|----------|----------|
| `-d` | Запуск в фоновом режиме (detached) |
| `--name qdrant` | Имя контейнера (можете выбрать своё) |
| `--restart unless-stopped` | Автоперезапуск при сбое и после перезагрузки сервера |
| `-p 6333:6333` | Проброс HTTP API порта |
| `-p 6334:6334` | Проброс gRPC порта |
| `-v qdrant_storage:/qdrant/storage` | Монтирование тома для данных |
| `qdrant/qdrant:latest` | Официальный образ из Docker Hub |

Docker автоматически скачает образ при первом запуске.

## Шаг 7. Проверка успешного запуска

### 7.1. Статус контейнера

```bash
docker ps
```

Ожидаемый результат:

```text
CONTAINER ID   IMAGE                  STATUS         PORTS
xxxxxxxxxxxx   qdrant/qdrant:latest   Up X seconds   0.0.0.0:6333-6334->6333-6334/tcp
```

### 7.2. Проверка ответа Qdrant

```bash
curl http://localhost:6333
```

Ожидаемый результат: JSON с информацией о версии:

```json
{"title":"qdrant - vector search engine","version":"1.x.x","commit":"..."}
```

## Шаг 8. Проверка доступности снаружи (опционально)

С вашего локального компьютера (не с сервера) выполните:

```powershell
curl http://<ПУБЛИЧНЫЙ_IP_СЕРВЕРА>:6333
```

Если ответ получен — порт открыт в файрволе/группе безопасности.  
Если нет — необходимо открыть порт 6333 в настройках облака/хостинга.

## Управление контейнером (основные команды)

| Действие | Команда |
|----------|---------|
| Посмотреть логи | `docker logs qdrant` |
| Логи в реальном времени | `docker logs -f qdrant` |
| Остановить | `docker stop qdrant` |
| Запустить снова | `docker start qdrant` |
| Перезапустить | `docker restart qdrant` |
| Удалить контейнер (после остановки) | `docker rm qdrant` |
| Принудительно удалить (даже если запущен) | `docker rm -f qdrant` |
| Зайти внутрь контейнера | `docker exec -it qdrant sh` |

## Альтернативные порты (если 6333 занят)

Если порт 6333 занят и освободить его невозможно, используйте другие порты:

```bash
docker run -d \
  --name qdrant \
  --restart unless-stopped \
  -p 6335:6333 \
  -p 6336:6334 \
  -v qdrant_storage:/qdrant/storage \
  qdrant/qdrant:latest
```

Тогда Qdrant будет доступен:

- **HTTP API:** порт 6335
- **gRPC:** порт 6336

## Обновление Qdrant до новой версии

```bash
# Остановить и удалить старый контейнер
docker stop qdrant
docker rm qdrant

# Скачать новую версию образа
docker pull qdrant/qdrant:latest

# Запустить контейнер заново (том с данными сохранится)
docker run -d \
  --name qdrant \
  --restart unless-stopped \
  -p 6333:6333 \
  -p 6334:6334 \
  -v qdrant_storage:/qdrant/storage \
  qdrant/qdrant:latest
```

Данные в томе `qdrant_storage` сохранятся.

## Что делать, если что-то пошло не так

### Ошибка: `Cannot connect to the Docker daemon`

**Решение:** Docker не запущен. Без sudo вы не сможете его запустить — обратитесь к администратору.

### Ошибка: `permission denied while trying to connect to the Docker daemon socket`

**Решение:** Выполните `newgrp docker` или переподключитесь по SSH. Если не помогает — вас нет в группе docker, нужен администратор.

### Ошибка: `address already in use` (порт занят)

**Решение:**

- `pkill -f qdrant` — убить бинарный процесс
- Или используйте альтернативные порты (см. выше)

### Ошибка: `Cannot start container ... port is already allocated`

**Решение:** Уже есть контейнер, использующий этот порт. Остановите его:

```bash
docker stop qdrant
docker rm qdrant
```
