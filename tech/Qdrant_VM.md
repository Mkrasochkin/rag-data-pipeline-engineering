# Инструкция: Запуск Qdrant без Docker на Yandex Cloud

## Кому подойдёт эта инструкция
Вы первый раз работаете с облачной ВМ, не знаете пароль от sudo, а Docker установлен, но нет прав на его использование. Эта инструкция поможет запустить Qdrant в обход всех этих проблем.

## Что такое Qdrant
Qdrant — это база данных для хранения и поиска векторов (векторная база данных). Она используется в проектах с искусственным интеллектом для быстрого поиска похожих объектов.

## Часть 1: Подготовка
### Шаг 1. Получите доступ к виртуальной машине
У вас должны быть:

- Публичный IP-адрес ВМ (например, 111.88.249.192)
- Логин пользователя (например, dashrim)
- SSH-ключ, добавленный в Yandex Cloud

Как подключиться по SSH из PowerShell (Windows):

```powershell
ssh dashrim@111.88.249.192

Что это значит:

- ssh — команда для защищённого подключения к удалённому серверу
- dashrim — ваше имя пользователя на сервере
- 111.88.249.192 — адрес сервера

### Шаг 2. Откройте порт 6333 в Yandex Cloud
Qdrant общается с внешним миром через порт 6333. По умолчанию он закрыт.

Как открыть:

- Зайдите в консоль Yandex Cloud: https://console.cloud.yandex.ru
- Virtual Private Cloud → Сети → ваша сеть (обычно default)
- Вкладка Группы безопасности → ваша группа (обычно default)
- Добавить правило:
  - Направление: Входящий трафик
  - Диапазон портов: 6333
  - Протокол: TCP
  - Назначение: CIDR
  - CIDR: 0.0.0.0/0 (доступ для всех)
  - Описание: Qdrant
- Нажать Сохранить

Зачем это нужно: Без открытого порта к Qdrant можно будет подключиться только изнутри ВМ, но не с вашего компьютера или от DS-команды.

## Часть 2: Установка и запуск Qdrant
### Шаг 3. Скачайте Qdrant
Подключитесь по SSH и выполните:

```bash
cd ~
wget https://github.com/qdrant/qdrant/releases/download/v1.16.2/qdrant-x86_64-unknown-linux-gnu.tar.gz
```

Что это значит:

- cd ~ — перейти в домашнюю папку пользователя
- wget [ссылка] — скачать файл из интернета
- Ссылка ведёт на готовую программу Qdrant для Linux (версия 1.16.2)

### Шаг 4. Распакуйте архив
```bash
tar -xzf qdrant-x86_64-unknown-linux-gnu.tar.gz
```

Что это значит:

- tar -xzf — распаковать сжатый архив .tar.gz
- После распаковки появится файл qdrant — это исполняемая программа

### Шаг 5. Создайте папку для хранения данных
```bash
mkdir -p ~/qdrant_storage
```

Что это значит:

- mkdir -p — создать папку, если её ещё нет
- Qdrant будет хранить все векторы и настройки в этой папке

### Шаг 6. Запустите Qdrant вручную (для проверки)
```bash
./qdrant --uri http://0.0.0.0:6333
```

Что это значит:

- ./qdrant — запустить скачанную программу
- --uri http://0.0.0.0:6333 — слушать порт 6333 на всех сетевых интерфейсах (чтобы доступ был и изнутри, и снаружи)

После запуска вы увидите логи работы Qdrant. Программа займёт текущее окно терминала.

### Шаг 7. Проверьте, что Qdrant работает
Не закрывая окно с Qdrant, откройте новое окно PowerShell и выполните:

```powershell
curl http://111.88.249.192:6333
```

Что это значит:

- curl — команда для отправки HTTP-запроса
- Если в ответ приходит JSON с "title":"qdrant - vector search engine" — всё работает!

Нажмите Ctrl+C в окне с Qdrant, чтобы остановить его (пока он работает вручную, а мы хотим запустить его в фоне).

### Шаг 8. Запустите Qdrant в фоновом режиме
```bash
nohup ./qdrant --uri http://0.0.0.0:6333 > qdrant.log 2>&1 &
```

Что это значит:

- nohup — запустить программу так, чтобы она продолжала работать после закрытия SSH-сессии
- > qdrant.log 2>&1 — записывать все сообщения (и ошибки) в файл qdrant.log
- & в конце — запустить программу в фоне (не занимать терминал)

### Шаг 9. Проверьте, что Qdrant работает в фоне
```bash
ps aux | grep qdrant
```

Что это значит:

- ps aux — показать все запущенные процессы
- | grep qdrant — отфильтровать только строки с qdrant
- Если видите строку с ./qdrant — программа работает в фоне

Проверьте доступность:

```bash
curl http://localhost:6333
```

## Часть 3: Автозапуск после перезагрузки ВМ
### Шаг 10. Настройте автозапуск через crontab
```bash
crontab -e
```

Что это значит:

- crontab — планировщик задач в Linux
- -e — редактировать список задач

При первом запуске система спросит, какой редактор использовать. Выберите 1 (nano).

В открывшемся файле добавьте строку в самый конец:

```text
@reboot /home/dashrim/qdrant --uri http://0.0.0.0:6333 > /home/dashrim/qdrant.log 2>&1 &
```

Что это значит:

- @reboot — выполнить команду при каждой перезагрузке системы
- Остальное — та же команда запуска Qdrant в фоне, что и в Шаге 8

Как сохранить и выйти из nano:

- Ctrl+O (сохранить)
- Enter (подтвердить)
- Ctrl+X (выйти)

### Шаг 11. Проверьте, что задача добавилась
```bash
crontab -l | grep qdrant
```

Должна показаться добавленная строка.
